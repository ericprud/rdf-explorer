# public/loaders/

This directory contains compiled ES module bundles of the built-in DataLoader
implementations.  Files here are generated by `scripts/build-loaders.mjs` and
are **not** committed to source control (add `public/loaders/*.js` to
`.gitignore`).

## Available loaders

| File                  | Source                          | Handles        |
|-----------------------|---------------------------------|----------------|
| `knows-parser.js`     | `src/lib/knows-parser.ts`       | `.txt`, `.knows`|

## How to generate

```bash
npm run build:loaders        # one-shot
npm run build:loaders --watch  # rebuild on source changes
```

Both files are also built automatically by `npm run build` (production) and
`npm run dev` (watch mode, runs alongside Vite).

## How to use

Drag one of these files onto the **Load** section title in the sidebar.
The app registers the loader and its drop-zone panel appears immediately.

This is exactly what a user with a custom loader would do with their own
compiled `.js` file — the built-in loaders are provided pre-compiled here
as an example and for production deploys where `loader-config.ts` is emptied.

## Writing a custom loader

See `src/lib/knows-parser.ts` for the minimal example.  The contract:

```js
// my-loader.js  (plain JS, no TypeScript needed)
export const parser = {
  name:        'My loader',
  description: 'Drop a .csv file',
  accepts:     ['.csv'],

  buildPanel(container, onTurtleChanged) {
    // Append a drop-zone to container.
    // Call onTurtleChanged(turtleString) after each parse.
    const zone = document.createElement('div')
    zone.textContent = 'Drop a .csv file'
    // ... wire up drag-and-drop and file input ...
    container.appendChild(zone)
  },
}
```

If you want the standard drop-zone widget without writing it yourself, compile
with `esbuild` and import `buildBasePanel` from `base-panel.ts`.
